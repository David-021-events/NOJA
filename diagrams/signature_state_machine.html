NOJA · v0.6 · §4.1 Signature Lifecycle State Machine

The Signature State Machine

Every signature exists in exactly one state. Transitions are signed events.

Standard transition
Recovery / return to Active
Exceptional path · requires justification

Superseded ≠ Revoked

Both exit Active and lead to Archived. The distinction is intent: Superseded is an orderly update — the successor was ready, no fallback was invoked. Revoked is withdrawal for cause — the fallback set engages immediately. Recording this in the state keeps orderly succession from being logged as an exceptional event.

The standard path is Drafted → Provisional → Active

Skipping Provisional is permitted but must be explicitly justified in the signature record. Provisional exists so the system can run in shadow mode — observable, logged, but taking no live action — before the signature goes fully active.
